# C++ Portfolio Project

## Project Summary
This project, CornerGrocer.cpp, was designed to analyze and process data from a file to provide meaningful output to the user. The program allows users to view item frequencies, search for specific items, and display data in a structured format. It solves the problem of organizing and analyzing grocery purchase data efficiently.

## What I Did Well
I did well in organizing my code and ensuring it was readable and easy to follow. I used clear variable names and structured my program logically. I also successfully implemented file input and output, which allowed the program to read and process real data.

## Areas for Improvement
I could improve my code by making it more efficient and modular. For example, I could create additional functions to reduce repetition and improve readability. I could also strengthen input validation to make the program more secure and user-friendly.

## Challenges and Solutions
One of the most challenging parts was working with file input and ensuring the data was read correctly. I overcame this by testing different inputs and reviewing course materials. Debugging also helped me identify and fix errors in the program.

## Transferable Skills
This project helped me develop skills in problem-solving, debugging, file handling, and writing clean code. These skills will be useful in future programming projects and in my career.

## Maintainability and Readability
I made this program maintainable by using comments and organizing the code into clear sections. This makes it easier to update and modify in the future.





# CS 230 – The Gaming Room Software Design Document

## Briefly summarize The Gaming Room client and their software requirements.

The client, The Gaming Room, wanted to expand their existing Android game, Draw It or Lose It, to multiple platforms. The game allows teams to guess images that are gradually revealed over time. They needed a software design that would support multiple users, teams, and game sessions while maintaining unique game names and efficient performance across different operating systems.

## What did you do particularly well in developing this documentation?

I did a good job analyzing the client's requirements and comparing different operating platforms. I clearly explained the advantages and disadvantages of each platform and provided recommendations that aligned with the client's goals for scalability, performance, and future growth.

## What about the process of working through a design document did you find helpful when developing the code?

Creating the design document helped me organize my thoughts before implementation. It allowed me to identify requirements, system constraints, and architectural decisions early in the process. This planning provided a clear roadmap for development.

## If you could choose one part of your work on these documents to revise, what would you pick? How would you improve it?

If I could revise one part of the design document, I would provide more detailed diagrams and technical explanations of the system architecture. Adding more visual representations would improve communication among developers and stakeholders.

## How did you interpret the user's needs and implement them into your software design? Why is it so important to consider the user's needs when designing?

I interpreted the user's needs by reviewing the client's requirements and focusing on functionality, scalability, and usability. I incorporated these needs into the design by recommending a cloud-based architecture, supporting multiple concurrent game sessions, and ensuring data consistency. Considering user needs is important because software is ultimately built to solve user problems and provide a positive experience.

## How did you approach designing software? What techniques or strategies would you use in the future to analyze and design a similar software application?

I approached software design by gathering requirements, analyzing possible solutions, evaluating operating platforms, and selecting the best architecture for the project. In the future, I would continue using requirements analysis, UML modeling, design documentation, and stakeholder feedback to guide the development process.
